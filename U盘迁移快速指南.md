# OpenClaw U盘迁移快速指南

## 🎯 迁移目的
1. **扩展记忆空间**：从小U盘（31GB）迁移到大U盘（4TB）
2. **存储项目文件**：为未来合作项目预留空间
3. **硬件升级**：更换更大容量的U盘

## 🔄 硬件变更说明
- **当前E盘**：小容量U盘（31GB），将被拿走
- **当前F盘**：大容量U盘（4TB），迁移后将变成**新的E盘**

## ⏰ 预计时间
- **数据复制**：10-30分钟（203MB数据）
- **配置更新**：5分钟
- **测试验证**：15分钟
- **总时间**：30-50分钟

## 🚀 快速操作步骤

### 阶段1：准备阶段（现在）
1. **确认F盘可用**：确保4TB U盘已连接为F盘
2. **备份提醒**：重要数据已备份（脚本会自动备份配置）
3. **停止OpenClaw**：确保OpenClaw没有正在运行

### 阶段2：执行迁移（运行脚本）
```cmd
cd /d "E:\openclaw-data\.openclaw\workspace"
usb_drive_migration.bat
```

### 阶段3：硬件切换
1. **安全移除当前E盘**（小U盘）
2. **等待系统重新识别**：F盘将自动变成E盘
3. **如果盘符没变**：可能需要手动分配盘符

### 阶段4：适应新环境
1. **运行适应脚本**：
   ```cmd
   E:\adapt_to_e_drive.bat
   ```
2. **测试启动**：
   ```cmd
   E:\start_openclaw_new_e.bat
   ```

### 阶段5：验证功能
1. 验证OpenClaw正常启动
2. 测试记忆访问
3. 检查所有工具功能

## 📋 关键文件位置

### 迁移后（新E盘，原F盘）
- 配置：`E:\openclaw-data\.openclaw\`

## ⚠️ 重要提醒

### 盘符变更
- 迁移后**F盘将显示为E盘**
- 所有路径引用会自动更新
- 如果盘符没自动变更，需要手动设置

### 数据安全
- 原E盘数据已完整复制
- 配置文件已备份
- 有完整的迁移日志

### U盘特性
- U盘读写速度可能较慢
- 建议定期备份到其他存储
- 避免频繁插拔

## 🔧 故障排除

### 问题1：F盘没有变成E盘
**解决**：
1. 打开"磁盘管理"
2. 找到4TB U盘
3. 右键 → 更改驱动器号和路径
4. 分配E盘盘符

### 问题2：OpenClaw无法启动
**解决**：
1. 检查配置文件路径
2. 运行适应脚本
3. 查看错误日志

### 问题3：记忆访问失败
**解决**：
1. 验证数据库文件完整性
2. 检查文件权限
3. 从备份恢复

## 📞 紧急回滚

### 如果需要回滚：
1. 重新插入原E盘（小U盘）
2. 确保显示为E盘
3. 直接使用原配置启动

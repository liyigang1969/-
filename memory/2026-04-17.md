## OpenClaw F盘独立系统创建

### 问题诊断历史
- 用户报告批处理文件闪退问题
- 发现OpenClaw配置错误：`openclaw.json`中存在无效的`"storage"`键
- 端口冲突问题：E盘和F盘OpenClaw尝试使用相同端口

### 解决方案实施
1. **创建F盘独立OpenClaw系统**：
   - 目录结构：`F:\OpenClaw_System\`
   - 程序目录：`F:\OpenClaw_System\program\` (包含openclaw.mjs)
   - 数据目录：`F:\OpenClaw_System\data\.openclaw\`
   - 启动脚本：`F:\OpenClaw_F_Drive.bat`

2. **避免端口冲突**：
   - F盘OpenClaw使用端口3005
   - E盘OpenClaw使用端口3000/3003
   - 独立环境变量配置

3. **修复批处理文件闪退**：
   - 创建HTA图形界面应用
   - 创建PowerShell脚本
   - 创建VBScript启动器
   - 使用`start`命令打开新窗口避免闪退

### 关键发现
- 批处理文件闪退原因为系统编码问题
- OpenClaw配置验证严格，不接受无效键
- 多盘OpenClaw实例需要独立端口配置

## C盘OpenClaw安装进展

### 安装方法选择
2026-04-17 17:45，决定使用npm安装C盘OpenClaw：
- Node.js版本：v22.22.2（已安装）
- 安装命令：`npm install -g openclaw@latest`

### 权限问题发现
执行安装时遇到PowerShell执行策略限制：
```
无法加载文件 C:\Program Files\nodejs\npm.ps1，因为在此系统上禁止运行脚本
```

### 解决方案
需要以管理员身份运行PowerShell并调整执行策略：
1. 以管理员身份打开PowerShell
2. 执行：`Set-ExecutionPolicy Bypass -Scope Process -Force`
3. 执行：`npm install -g openclaw@latest`

### 技术要点记录
1. **Windows PowerShell执行策略** - 默认限制脚本执行
2. **管理员权限需求** - 全局安装npm包需要管理员权限
3. **临时策略调整** - `-Scope Process`仅影响当前会话
4. **安装路径** - 全局安装到Node.js的全局模块目录

# 持续工作自动化系统 - 使用指南

## 系统概述
这是一个模拟人工回复继续机制的持续工作自动化系统，解决了AI助手缺乏持续思考和工作能力的问题。

## 核心特性
1. **持续思考** - 模拟人类不间断的思考过程
2. **自动化工作** - 自动处理工作队列中的任务
3. **状态跟踪** - 实时监控系统状态和性能
4. **思考日志** - 记录完整的决策和思考过程
5. **心跳集成** - 与OpenClaw心跳系统无缝集成

## 系统架构

### 主要组件
```
workspace/
├── workflow_system.md          # 系统设计文档
├── work_queue.json            # 工作队列（核心）
├── work_state.json            # 实时状态
├── HEARTBEAT.md              # 心跳集成配置
├── state_updater.js          # 状态更新工具
├── auto_worker.js            # 自动化工作脚本
├── thinking_logs/            # 思考日志目录
│   ├── index.json           # 日志索引
│   └── *.md                # 具体思考日志
└── 持续工作系统使用指南.md    # 本文档
```

## 快速开始

### 1. 添加新任务
编辑 `work_queue.json`，在 `tasks` 数组中添加新任务：
```json
{
  "id": "unique_task_id",
  "description": "任务描述",
  "priority": 数字,  // 越小优先级越高
  "status": "pending",
  "created_at": "ISO时间戳",
  "estimated_time": "预计耗时"
}
```

### 2. 启动自动化工作
```bash
# 启动自动化工作系统
node auto_worker.js start

# 查看系统状态
node auto_worker.js status

# 停止系统
node auto_worker.js stop
```

### 3. 手动更新状态
```bash
# 更新所有状态信息
node state_updater.js
```

## 工作流程

### 自动化工作流程
1. **任务发现** - 系统定期检查工作队列
2. **优先级排序** - 按优先级选择下一个任务
3. **思考模拟** - 模拟人类思考过程（1-3秒）
4. **任务执行** - 处理任务逻辑
5. **状态更新** - 更新任务状态和系统状态
6. **日志记录** - 创建思考日志
7. **继续循环** - 检查下一个任务

### 心跳集成流程
1. OpenClaw心跳触发（每5分钟）
2. 检查HEARTBEAT.md中的配置
3. 执行工作队列检查逻辑
4. 如有任务则开始处理
5. 更新系统状态

## 配置选项

### 工作队列设置 (work_queue.json)
```json
"settings": {
  "auto_continue": true,           // 自动继续
  "check_interval_minutes": 5,     // 检查间隔
  "max_concurrent_tasks": 1,       // 最大并发任务
  "log_thinking": true            // 记录思考
}
```

### 自动化工作设置 (auto_worker.js)
- 检查间隔: 5分钟
- 思考时间: 1-3秒（随机）
- 工作时间: 2-5秒（随机，可调整）

## 监控和调试

### 查看实时状态
```bash
# 查看工作队列状态
cat work_queue.json | jq '.tasks[] | select(.status=="pending")'

# 查看系统状态
cat work_state.json | jq '.system'

# 查看性能指标
cat work_state.json | jq '.performance'
```

### 查看思考日志
```bash
# 列出所有思考日志
ls -la thinking_logs/*.md

# 查看最新思考日志
tail -f thinking_logs/$(ls -t thinking_logs/*.md | head -1)
```

## 故障排除

### 常见问题

1. **任务不执行**
   - 检查任务状态是否为 "pending"
   - 检查优先级设置
   - 检查自动化工作脚本是否运行

2. **状态不更新**
   - 检查 work_state.json 文件权限
   - 检查 state_updater.js 是否正常工作
   - 查看控制台错误信息

3. **思考日志不创建**
   - 检查 thinking_logs/ 目录权限
   - 检查日志文件名是否冲突
   - 查看磁盘空间

4. **系统性能问题**
   - 调整检查间隔时间
   - 减少并发任务数
   - 优化任务处理逻辑

### 日志文件位置
- 系统日志: 控制台输出
- 思考日志: thinking_logs/*.md
- 状态历史: work_state.json（实时状态）
